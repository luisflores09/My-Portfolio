Reflection on my learning

The biggest shift for me has been moving from “getting it working” to “getting it working well.” I’ve learned to slow down at the right moments—clarify requirements, choose simpler designs, and make changes in a way that keeps the codebase easy to extend.

  • Engineering fundamentals: stronger TypeScript patterns, component architecture, and cleaner separation of concerns
  • Reliability: more deliberate testing and better debugging habits (repro steps, logging, and isolating variables)
  • Security mindset: competed in a cybersecurity tournament focused on API vulnerabilities and the OWASP Top 10, reinforcing secure-by-default habits (auth, access control, and input validation)
  • Delivery: improved planning and iteration—shipping smaller slices and polishing based on feedback
  • Using AI responsibly: I use AI tools to speed up scaffolding and refactors, but I validate outputs and keep standards high (readability, correctness, and security)
  • Communication: clearer status updates, trade-offs, and documentation so others can follow the work
